Seccionado

Para poder desarrollar la página web con el menor número de inconvenientes posibles, se decidió seccionar, catalogar y documentar todos los cambios en cada uno de los archivos que se visualizan en GitHub Pages.

El seccionado se llevó a cabo de la siguiente manera:

HTML - Comentarios que indiquen el claro seccionado del código.

CSS - Índice de contenido (indispensable) y comentarios que indiquen el claro seccionado del código.

JavaScript - Comentarios que indiquen el claro seccionado y funcionamiento del código.

Nota Importante: Cualquier subsección de los códigos clasificados como importantes, también se catalogarán como tal en código y en bitácora.

Responsividad

La página se creó con responsividad exitosa, con funcionalidad y visibilidad correcta en el 100% de los dispositivos móviles en los que se probó. Para cambios o adiciones futuras, la resposividad se catalogará de forma aislada al resto de los archivos CSS.


Como principio, la funcionalidad de la página web no debe opacar su responsividad, ni viceversa.

Buenas Prácticas

Como convención, y por buenas prácticas, se decidió trabajar con las siguientes pautas en mente:

🙤 1. Escribir Código Limpio


🙪 Nombres Claros: Nombramiento de variables, clases, ids, funciones, páginas, imágenes, etc.


🙪 Comentarios:Se documenta con breves descripciones lo que se trabaja para que todo el equipo entienda.


🙤 2. Control de Versiones


🙪 Git:Se usa Git como un control de versiones para gestionar cambios, se hacen commits frecuentes con mensajes claros.


🙤 3. Responsividad y Accesibilidad


🙪 Diseño responsivo:Nos aseguramos que la aplicación sea usable en cualquier dispositivo


🙪 Accesibilidad:Nos aseguramos que todos los usuarios puedan interactuar con nuestra ecommerce


🙤 4. Testing


🙪 Pruebas Unitarias: Se escriben pruebas unitarias para asegurar que cada parte del código funciona correctamente


🙪 Pruebas de Integración:Verificamos que diferentes partes del sistema funcionen juntas.


🙤 5. Colaboración


🙪 Revisión de código:Se participa en revisiones de código para compartir conocimientos y mejorar la calidad del código


🙪 Comunicación:Se prioriza la buena comunicación para asegurar que todos tenemos el mismo objetivo
